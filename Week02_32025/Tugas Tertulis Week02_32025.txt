2. A& B -> error yang terjadi adalah NumberFormatException: empty String
		   Terjadi karena nilai String adalah null karena tidak diberikan input. Karena hasilnya null, nilai tidak dapat diconvert jadi string dan juga diparse. Hal ini dapat diresolve dengan membuat condition terlebih dahulu untuk mengecek apakah nilai op1 dan op2 mempunyai nilai null atau tidak. Jika nilai adalah null, nilai pada input tersebut dapat diberikan nilai 0, ataupun diberikan popup atau warning yang akan memberitahu user untuk memberikan input.
	C -> Hasil nilai adalah infinity. Dalam java normalnya, akan terjadi ArithmeticException karena hasil dari bilangan dibagi dengan 0 adalah tidak terhingga. Dikarenakan hasil nilai adalah infinity, hal ini menunjukkan bahwa sudah ada exception handling untuk masalah tersebut pada android studio.

3. Debugging point ditaruh pada point 45 dan 54. Dari dua point tersebut, didapatkan bahwa program akan berhenti pada debugging point. Nilai yang didisplay adalah semua variable yang sudah diproses sebelum line yang diberikan debugging point tersebut. Hal ini berarti line tersebut belum dirun. Untuk merun line tersebut satu per satu, step over button akan dipencet. Step Into digunakan untuk masuk ke method internal yang digunakan oleh android studio. Step Out digunakan untuk kembali ke baris semula dimana step into dilakukan. Resume digunakan untuk menjalankan kembali program. Jika Resume button ditekan pada point 45, program akan berhenti pada line 54 dikarenakan adanya debugging point kedua.
